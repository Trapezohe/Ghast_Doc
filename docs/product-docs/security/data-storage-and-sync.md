# 数据存储与同步边界

## 本页说明

本页说明 Ghast AI 的主要数据分别放在哪里、哪些内容默认只保留在本地，以及远程同步该怎样理解。

## 当前主要数据放在哪里

| 数据类型 | 当前主要位置 |
| --- | --- |
| 钱包、会话、设置、远程控制状态等扩展数据 | 浏览器扩展本地存储 |
| 长期记忆与记忆管理 | 扩展记忆系统 / Memory 页面 |
| 工作区文件 | 你主动连接的本地文件夹 |
| Companion 本地配置 | `~/.trapezohe` |
| 可选的远程记忆同步数据 | 仅在你启用相关同步后参与 |

对普通用户来说，最重要的结论是：不同类型的数据并不在同一个地方，也不会因为安装了 Companion 就自动全部进入同一条同步路径。

## 默认不应混淆的几件事

为了正确理解数据边界，建议直接记住下面三点：

- 工作区文件不等于 Ghast AI 的核心记忆。
- Companion 本地配置不等于浏览器扩展里的本地数据。
- 远程同步存在，不代表所有本地数据都会自动同步到远端。

## 远程同步该怎么理解

当前远程同步更适合被理解成一项可选能力，而不是默认必须开启的基础路径。

对普通用户而言，更稳妥的理解方式是：

- 只有在你明确启用相关同步后，远程同步才会参与。
- 远程同步主要围绕需要同步的那部分内容展开，而不是复制整套本地环境。
- 如果你还没有明确同步需求，就不需要把它当成第一次使用时的重点。

## 什么时候你才需要深入关心这页

一般只有在下面这些情况下，这页才会真正变得重要：

- 你正在管理长期记忆。
- 你需要判断某类数据到底保存在本地哪里。
- 你在处理同步、迁移或恢复相关问题。

如果你只是日常使用，记住“不同数据有不同位置，远程同步不是默认全开”通常就足够。

Ghast AI 当前把扩展数据、长期记忆、工作区文件和 Companion 本地配置分散在不同位置管理；远程同步属于可选能力，而不是默认全量同步路径。对普通用户而言，更适合先把它理解为“按数据类型分别管理、按需同步”的边界模型。
