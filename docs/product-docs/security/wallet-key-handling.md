# 钱包密钥处理

## 本页说明

本页说明 Ghast AI 当前如何处理钱包私钥和助记词，以及这些处理方式对普通用户意味着什么。

对大多数用户来说，最重要的结论是：钱包由扩展本地管理，默认会自动上锁，不会因为安装 Companion 而变成另一套独立存储路径。

## 先看结论

当前钱包边界可以概括为三句话：

- 钱包信息主要保存在扩展本地
- 钱包不是长期明文开放状态
- 默认会在一段时间后自动重新锁定

这意味着，钱包在当前产品里是一条单独的敏感边界，而不是附着在普通设置上的次要信息。

## 创建、导入和解锁时会发生什么

当你创建钱包、导入私钥或导入助记词时，当前系统会先把这些敏感信息以本地加密方式保存，而不是以可直接读取的形式长期留在本地存储中。

当你解锁钱包后，系统只会在当前使用过程中临时持有必要的可用状态；当自动锁定生效，或你手动锁定后，这种可用状态会被收回。

## 默认保护是什么

当前最重要的默认保护是自动上锁。默认时间为 10 分钟。

这条默认设置的意义非常直接：

- 你不需要一直手动记得锁钱包
- 钱包不会因为一次解锁就长期保持在可用状态
- 日常使用里，即使你短时间离开，也不会天然处于长期解锁状态

## Companion 与钱包的关系

Companion 会扩展 Ghast AI 的本地能力，但它不是钱包的主要存储位置。

对普通用户来说，可以直接记住这一点：

安装 Companion 不等于把钱包交给 Companion 管理。钱包边界仍然主要在扩展本地这一侧。

## 哪些做法会明显放大钱包风险

下面这些做法都会让钱包边界明显变宽：

- 长时间保持钱包解锁
- 把自动锁定设为永不自动锁
- 在不必要时频繁导出敏感信息
- 把“本地加密保存”误解成“任何时候都完全无风险” 

尤其要注意的是：默认自动上锁之所以重要，正是因为钱包在解锁使用期间仍然属于更高敏感状态。

## 普通用户的使用建议

如果你的目标是保持当前默认边界，建议这样使用：

1. 保持默认自动上锁，除非你有非常明确的长期需求。
2. 只在确实需要时再解锁钱包。
3. 不把导出私钥或助记词当成日常操作。
4. 把钱包看成一条独立敏感边界，而不是普通设置项。

Ghast AI 当前将钱包作为独立的本地敏感边界处理：钱包信息主要保存在扩展本地，并通过本地加密和默认自动上锁来降低长期暴露风险。用户一旦主动延长解锁时间或关闭自动上锁，这条默认边界就会随之变宽。
