# Chrome 权限与主机权限

## 本页说明

Ghast AI 作为浏览器扩展，部分能力依赖浏览器权限。本页不从开发角度解释权限清单，而是从使用角度说明：这些权限为什么存在、默认大致覆盖哪些范围，以及什么时候你需要对权限变化保持警惕。

## 先看结论

Ghast AI 当前不是一个“对所有网站默认同等生效”的扩展。即使扩展具备一定浏览器权限，真正会参与页面上下文的站点范围仍然是有限的。

对普通用户来说，最重要的是理解两件事：

- 扩展确实需要一些浏览器权限才能正常工作
- 有权限不等于所有网页都会被默认读取或采集

## 这些权限主要用来做什么

| 权限类别 | 对用户意味着什么 |
| --- | --- |
| 本地存储 | 保存设置、记忆、钱包相关本地状态 |
| 当前标签页与页面访问 | 在受支持页面中读取当前页面上下文 |
| 侧边栏能力 | 让 Ghast AI 作为浏览器侧边栏使用 |
| 定时与通知 | 支持提醒、心跳和部分后台任务 |
| 页面注入能力 | 在受支持站点中启用页面理解能力 |
| 本地配对能力 | 发现并连接 Companion |
| 登录能力 | 支持 Google 登录 |

你可以把这些权限理解为“让扩展成为可用产品所需的浏览器能力”，而不是“默认打开所有网页访问”。

## 当前大致覆盖哪些站点范围

当前主机权限主要围绕以下类别展开：

- 0G 相关站点
- 常见区块浏览器
- 常见 DeFi 协议站点
- X / Twitter
- Discord
- Google 相关服务
- Trapezohe 自有服务
- 技能与搜索相关站点
- 本地 `localhost` / `127.0.0.1`

这意味着 Ghast AI 当前更像一个围绕 Web3 和本地能力组织的扩展，而不是面向全网任意站点的通用采集扩展。

## 需要单独注意的地方

当前还存在一类“按需授权”的更宽访问范围。它不是默认一直打开，但如果你主动授权，扩展可访问的网页范围会明显变宽。

对普通用户来说，理解成下面这句话就够了：

如果你没有明确需要，就尽量保持默认权限范围，不要额外放宽到更广的网站访问范围。

## Google 登录当前意味着什么

Ghast AI 当前的登录能力只围绕基础账号识别展开，用于完成登录和账号状态识别，不应理解为“读取你的整个 Google 账户内容”。

## 这页对使用手册的意义

阅读本页后，你应该形成的判断是：

- Ghast AI 确实需要浏览器权限才能工作
- 这些权限有明确用途，不是为了无差别读取所有网页
- 页面上下文能力仍然有单独的支持范围
- 如果主动放宽权限，你获得的是更大能力范围，同时也是更大的边界范围
