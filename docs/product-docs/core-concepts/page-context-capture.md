# 页面上下文采集模型

## 本页说明

本页说明 Ghast AI 如何把“你正在看的页面”转成可用上下文，以及普通用户应怎样理解这条能力边界。

## 先看结论

对普通用户来说，这条能力最重要的结论只有三点：

- 页面上下文只在当前支持的页面范围内生效。
- 这项能力可以按设置选择开启、关闭或弱化。
- 它的目标是增强页面场景理解，而不是对全网页默认开放采集。

## 页面上下文到底是什么

页面上下文可以直接理解成：Ghast AI 不只看你输入的问题，也会在支持页面里结合你当前正在看的内容来理解意图。

因此，这项能力的重点不是“读取越多越好”，而是“在合适范围内，把当前页面转成更可用的上下文”。

## 什么时候它会生效

一般只有在下面这些条件成立时，这条能力才会生效：

- 当前页面属于支持范围。
- 相关页面能力当前没有被关闭。
- 你当前的使用方式确实需要围绕页面提问。

如果这些条件不成立，聊天仍然可以继续，只是不会获得同等的页面上下文增强。

## 普通用户该怎样控制它

这条能力更适合被理解成一项可控制的页面增强能力。对普通用户来说，更实用的控制原则是：

- 想让 Ghast AI 围绕当前页面工作时，就保持相关页面能力开启。
- 只想纯聊天，或暂时不希望页面参与时，就关闭或弱化这类能力。

## 不应怎样理解这条边界

- 不要把页面上下文理解成对所有网站默认同等开放。
- 不要把页面上下文理解成无条件读取整页一切内容。
- 不要把“某页没有同等效果”直接判断为异常，很多时候只是页面不在支持范围内。

Ghast AI 当前通过受支持页面中的页面上下文能力，把“当前正在看的页面”转化为可用上下文。对普通用户而言，这是一项可增强页面场景理解、但并非对全网页默认开放的能力。
