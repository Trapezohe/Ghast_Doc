# Skills 与 MCP 模型

## 本页说明

本页帮助你从用户角度区分 Skills 和 MCP 分别负责什么，以及什么时候你只需要关心 Skills，什么时候才需要继续进入 MCP。

## 先分清两层

| 层级 | 对普通用户更实用的理解 |
| --- | --- |
| Skills | 让 Ghast AI 按更稳定的方法完成某类任务 |
| MCP | 为部分技能或高级本地场景补充更深的工具能力 |

因此，它们经常一起出现，但并不是同一件事。

## 为什么普通用户应先从 Skills 开始

对大多数用户来说，先进入 Skills 就已经足够：

- 你可以先导入自己真正需要的技能。
- 你可以先观察某个技能是否已经满足任务需求。
- 你不需要一开始就理解 MCP 的更深本地能力层。

只有当某个技能明确需要更深的本地工具支持时，MCP 才会真正变得重要。

## 什么时候才需要继续关心 MCP

一般只有在下面这些情况下，才值得继续进入 MCP：

- 你已经安装了 Companion。
- 你已经连接了工作区，并跑通基础本地路径。
- 某个技能需要额外本地工具，单靠导入本身还不够。

如果这些前提还没有成立，通常不应把 MCP 当作第一次上手的重点。

## 更稳妥的使用顺序

1. 先导入技能。
2. 先验证技能是否已经满足你的任务。
3. 如果确实需要更深本地工具支持，再继续配置 MCP。

这样做更符合普通用户的使用节奏，也更能避免一开始就把本地能力路径做得过重。

在 Ghast AI 当前产品中，Skills 用于扩展助手的任务方法，MCP 用于支撑更高级的本地工具能力。对普通用户而言，推荐先从技能能力开始理解，并在明确需要更深本地工具支持时，再进一步关注 MCP。
