# 远程控制模型

## 本页说明

本页说明 Ghast AI 中“远程控制”应该怎样理解、什么用户才需要开启，以及在什么前提下再进入远程通道会更合适。

## 先判断你现在需不需要它

| 使用需求 | 是否建议现在开启远程控制 |
| --- | --- |
| 主要在浏览器侧边栏里使用 | 暂时不需要 |
| 离开浏览器后仍希望继续通过 Telegram 找到助手 | 可以考虑 |
| 扩展基础路径还没有跑通 | 先不要 |
| 还没有想清楚远程入口会带来额外边界 | 先不要 |

对普通用户而言，远程控制始终属于按需接入的附加入口，而不是默认主路径。

## 开启前先确认什么

在你进入远程通道之前，建议先确认：

1. 扩展已经正常安装、登录并完成激活。
2. 侧边栏内的基础使用已经稳定。
3. 你已经理解远程入口和本地直接操作不是同一条边界。
4. 你确实需要在离开浏览器后继续使用 Ghast AI。

如果这些前提还没成立，通常不应该优先开启远程控制。

## 更稳妥的接入顺序

对普通用户来说，更合适的顺序通常是：

1. 先把浏览器侧边栏作为主入口用顺。
2. 再在 `设置 > 远程控制` 中配置 Telegram。
3. 先完成 Bot 配置和聊天关联。
4. 最后再测试远程交互和审批路径。

这样做的目的，是先把主路径稳定下来，再引入新的远程边界。

## 什么情况下不应把问题先归到远程控制

如果你当前连登录、激活、钱包或主界面基础使用都还没有稳定，那么问题通常还不属于远程控制本身。

更稳妥的做法，是先把主入口路径跑通，再进入远程通道。

Ghast AI 当前的远程控制属于按需接入的附加入口，而不是默认主使用路径。对普通用户而言，推荐先以浏览器扩展作为主入口；只有在明确存在远程交互需求时，再继续配置远程通道。
