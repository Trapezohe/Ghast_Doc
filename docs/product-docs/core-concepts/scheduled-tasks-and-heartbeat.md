# 定时任务与 Heartbeat

## 本页说明

本页说明定时任务和 Heartbeat 分别适合解决什么问题，以及普通用户该怎样在这两条持续能力路径之间做选择。

## 先按需求选路径

| 你的需求 | 更适合的能力 |
| --- | --- |
| 在固定时间提醒或执行一件事 | 定时任务 |
| 持续巡视某个主题，无事时保持安静 | Heartbeat |
| 只想先跑通最简单的持续能力 | 先用定时任务 |
| 已经有长期观察清单，希望按节奏检查 | 再考虑 Heartbeat |

对普通用户而言，最实用的区别是：定时任务更像“按计划去做”，Heartbeat 更像“按计划去看，并在有必要时再提醒或行动”。

## Heartbeat 应该怎么理解

Heartbeat 当前更适合被理解成一条持续巡视路径：它会定期读取 `heartbeat.md` 清单，让 AI 判断是否需要提醒或继续行动；如果没有需要处理的内容，就保持安静。

因此，Heartbeat 的重点不是“每次都要有消息”，而是“长期看着，只有需要时再出声”。

## 更稳妥的开始方式

对大多数用户来说，更推荐按下面顺序进入：

1. 先从一个简单、明确的定时任务开始。
2. 先确认自己真的需要持续能力。
3. 当你发现自己需要长期关注同一类事项时，再进入 Heartbeat。

这样做，会比一开始就把所有持续需求都做成长期监控更稳妥。

## 不要先怎么做

- 不要把所有提醒都直接做成 Heartbeat。
- 不要把 Heartbeat 的安静状态直接理解成故障。
- 不要在基础路径还没稳定前，就先把持续能力做得过重。

Ghast AI 当前将持续能力分为定时任务与 Heartbeat 两类：前者用于按时间执行，后者用于按时间检查并在有必要时再提醒或行动。对普通用户而言，建议先从简单定时任务开始，在明确存在长期关注需求时再使用 Heartbeat。
