# Extension 与 Companion 的工作方式

## 本页说明

本页帮助你理解 Ghast AI 的两个核心组成部分：浏览器扩展和 Companion。

对普通用户来说，这页最重要的作用不是介绍内部实现，而是回答两个实际问题：

- 不安装 Companion 时，我能用到什么程度？
- 安装 Companion 后，产品为什么会变得更强？

## 可以先这样理解两者关系

| 组件 | 对用户来说是什么 | 主要负责什么 |
| --- | --- | --- |
| Extension | 你每天直接使用的主界面 | 聊天、记忆、技能、钱包、页面上下文、设置 |
| Companion | 扩展之外的本地能力层 | 工作区、本地能力、更强自动化和高级本地能力 |

换句话说：

- 扩展负责“你在浏览器里看到和直接使用的部分”
- Companion 负责“浏览器本身做不到、但本地环境可以补上的部分”

## 只使用扩展时，你已经能做什么

即使不安装 Companion，当前 Ghast AI 仍然可以完成很多基础使用路径，包括：

- 登录和激活账号
- 在侧边栏中聊天
- 管理记忆
- 管理技能
- 管理钱包
- 选择模型
- 在支持站点上结合当前页面使用

这意味着，扩展本身已经是一套可独立使用的主产品，而 Companion 不是“没有就完全不能用”的前置条件。

## 安装 Companion 后，产品为什么会变强

浏览器扩展天生更适合做界面、页面理解和轻量交互；而涉及工作区、本地文件、本地命令和更强的本地能力时，就需要 Companion 来补上。

因此，安装 Companion 后，Ghast AI 会多出一层真正属于本机的能力面，例如：

- 围绕工作区的本地使用能力
- 更强的本地任务能力
- MCP 和更高级的自动化路径
- 诊断、自检和本地修复相关能力

对用户来说，最简单的理解方式是：

扩展决定“你怎么用”，Companion 决定“你最多能把产品用到什么深度”。

## 什么时候建议安装 Companion

如果你只是想先体验对话、记忆、技能和页面理解，当前可以先只安装扩展。

如果你有下面这些需求，就建议尽早安装 Companion：

- 希望围绕本地工作区使用 Ghast AI
- 希望获得更强的本地能力
- 希望使用 MCP 或更深的自动化路径
- 希望把 Ghast AI 用作更完整的本地助手，而不仅仅是浏览器侧边栏

## 两者之间的边界

理解 Extension 和 Companion 的关系时，最重要的是不要把它们当成同一个东西。

你可以把边界记成下面三句话：

- 扩展是主入口
- Companion 是本地能力层
- 是否安装 Companion，决定的是能力深度，而不是是否能开始使用

这也是为什么手册里会把两者分成独立章节来写。

## 普通用户的推荐路径

如果你不确定该怎么开始，建议按下面顺序使用：

1. 先安装扩展并完成登录。
2. 先熟悉聊天、记忆、钱包和页面理解。
3. 当你确实需要工作区或更强本地能力时，再安装 Companion。
4. 安装 Companion 后，先保持默认更收敛的使用方式，再逐步放开能力范围。

Ghast AI 当前由扩展和 Companion 两部分共同组成。扩展是用户的主使用入口，Companion 是扩展之外的本地能力层。对普通用户而言，扩展已经足以完成基础使用；而 Companion 的价值，在于把产品从“浏览器助手”扩展为“可连接本地环境的更完整助手”。
