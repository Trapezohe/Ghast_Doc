# 记忆模型

## 本页说明

本页说明 Ghast AI 中的“记忆”该怎样理解，以及它和普通聊天记录、本地工作区文件为什么不是同一回事。

## 先分清三类内容

| 内容 | 更适合放在哪里 |
| --- | --- |
| 一次性的对话往来 | 当前聊天过程 |
| 需要长期保留的稳定上下文 | Memory |
| 项目源码、文档和本地文件 | 工作区 |

对普通用户而言，记忆的作用不是“保存一切”，而是帮助 Ghast AI 长期保留真正重要的上下文。

## 记忆更适合承载什么

记忆更适合承载下面这类内容：

- 与你长期相关的稳定信息。
- 在多次使用中都需要保留的重要上下文。
- 适合长期整理、迁移或备份的内容。

如果一段信息只在当前聊天里短暂有用，它通常不需要被当成长期记忆来管理。

## 为什么它不是工作区文件

工作区文件属于你主动连接的本地目录；记忆属于 Ghast AI 自己的长期上下文层。

这也是为什么产品把 Memory 做成独立入口，而不是让你通过普通文件路径去管理它。

## 普通用户什么时候才需要深入关心

如果你只是第一次体验产品，可以先把记忆理解成“助手会长期保留一部分重要内容”。

当你开始：

- 长期使用 Ghast AI。
- 希望保留稳定的个人上下文。
- 需要整理、备份或迁移长期内容。

这时再深入管理记忆会更有意义。

Ghast AI 当前的记忆模型用于承载长期上下文，而不是替代普通聊天记录或本地工作区文件。对普通用户而言，更适合将其理解为产品长期保留重要上下文的一层能力，并在长期使用阶段再进一步整理和管理。
