# Wallet and Model Funding

## Overview

This page explains the relationship between the wallet and model funding in Ghast AI, and the order in which most users should enter this funding path during first use.

## Separate the two funding layers first

| Layer | What it means for users |
| --- | --- |
| Wallet | Your local funding entry point |
| Model funding | The available amount prepared for a specific model |

Once those two layers are clear, recharge, top-up, and refund behavior become much less confusing.

## Recommended order

The steadier order is usually:

1. Establish the local funding entry point through the wallet.
2. Choose one primary model on the model page.
3. Prepare initial funds for that model.
4. After real usage begins, add or adjust funds based on actual consumption.

That is usually a better daily-use pattern than funding many models at once from the start.

## When you do not need to recharge yet

If you are still learning the product, browsing pages, or completing the base setup path, there is usually no need to prepare funds for multiple models immediately.

The steadier approach is to pick one primary model, prepare enough funds for that model to begin using it, and decide on further top-ups only after real usage starts.

## How to interpret refunds

A refund is better understood as reclaiming the portion that is currently refundable, not as withdrawing every visible amount immediately.

For most users, the important points are:

- Refundable amount depends on the current refundable scope.
- The refundable amount does not necessarily equal the full balance visible on the page.
- A recently recharged portion may not enter the refundable range immediately.

So when you see a difference, the better first interpretation is a difference in current stage rather than an automatic assumption that something is wrong.

Ghast AI currently treats wallet funds and model funds as two separate layers. The wallet establishes the local funding entry point, while model funding supports actual model use. For most users, the steadier path is to complete wallet setup first and then prepare and manage available funds gradually around one primary model.
