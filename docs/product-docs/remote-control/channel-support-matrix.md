# 通道支持矩阵

## 本页说明

本页用于说明当前远程通道各自处于什么状态，以及普通用户现在应把哪些通道当成正式路径，哪些不应视为默认入口。

## 先看当前结论

对普通用户而言，当前应按下面的原则理解这张表：

- Telegram 是当前手册中的标准远程入口。
- 代码里已实现但当前未作为主 UI 路径公开的通道，不应当成普通用户默认功能使用。
- 计划中或当前范围外的通道，不应被理解为“现在就能直接配置”。

## 当前矩阵

| 通道 | 当前状态 | 手册中的写法 | 普通用户应怎样理解 |
| --- | --- | --- | --- |
| Telegram | 已作为正式远程入口说明 | 主路径 | 可按当前手册直接配置和使用 |
| Discord | 代码中已实现 | 已实现，但不作为当前主 UI 入口 | 不按普通用户默认路径使用 |
| Slack | 代码中已实现 | 已实现，但不作为当前主 UI 入口 | 不按普通用户默认路径使用 |
| Google Chat | 计划中 | 计划中 | 当前不作为可直接配置通道 |
| IRC | 计划中 | 计划中 | 当前不作为可直接配置通道 |
| WhatsApp | 当前范围外 | 当前不支持 | 不应视为可用通道 |
| Signal | 当前范围外 | 当前不支持 | 不应视为可用通道 |
| BlueBubbles | 当前范围外 | 当前不支持 | 不应视为可用通道 |
| iMessage | 当前范围外 | 当前不支持 | 不应视为可用通道 |

## 当前写文档时的处理原则

为了保持手册口径稳定，当前远程通道文档遵循下面两条原则：

- 面向普通用户的操作说明，只按 Telegram 主路径编写。
- 代码里已有但当前 UI 未公开的通道，只记录状态，不写成默认操作指南。

当前远程通道中，Telegram 是 Ghast AI 面向普通用户的标准远程入口。其他通道如 Discord、Slack 虽然在代码中已有实现，但当前不作为主 UI 路径公开；计划中或当前范围外的通道，也不应被视为普通用户现在可直接配置的正式入口。
