# 审批流

## 本页说明

本页说明在远程使用 Ghast AI 时，为什么有些动作需要审批，以及普通用户应怎样理解这条流程。

## 为什么远程路径需要审批

远程控制并不是把侧边栏原样搬到另一个通道里。只要远程入口参与了产品使用，系统就需要在更敏感的动作上保留确认步骤。

对普通用户来说，审批流的意义可以直接理解成：在需要更谨慎处理的时候，系统不会默认把所有动作都直接执行掉。

## 这条流程对用户意味着什么

它主要带来三件事：

- 远程动作不是默认无限制执行。
- 某些更敏感的操作会要求再次确认，或以更谨慎的方式处理。
- 远程使用仍然保留边界，而不是完全等同于本地直接操作。

## 正常使用时应怎样预期

如果你已经开启远程控制，应当预期下面这些表现都属于正常情况：

- 某些动作不会立刻一步完成。
- 远程使用体验可能和侧边栏本地操作不完全一样。
- 当系统要求确认时，这首先是边界控制，而不是配置失败。

## 更稳妥的使用方式

对普通用户来说，更稳妥的方式是：

1. 先只在确实需要时开启远程控制。
2. 先用简单、低风险的远程操作确认通道已经接通。
3. 接受“更敏感的动作可能需要审批”是这条路径的正常组成部分。

Ghast AI 当前的远程审批流用于在远程入口下保留必要的确认步骤。对普通用户而言，它的意义不是增加复杂度，而是在远程使用场景中为更敏感的动作保留清晰边界。
