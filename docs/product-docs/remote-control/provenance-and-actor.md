# 远程执行者与来源标记

## 本页说明

本页说明 Ghast AI 为什么要区分“这次请求来自哪里”，以及这种区分对普通用户有什么实际意义。

## 对普通用户更实用的理解方式

你不需要记住底层字段，只需要知道：系统会尽量区分这次请求到底来自哪一类入口。

| 请求来源 | 对普通用户意味着什么 |
| --- | --- |
| 本地聊天 | 你在侧边栏里直接发出的请求 |
| 远程渠道 | 你通过 Telegram 等远程通道发出的请求 |
| 定时 / Heartbeat | 系统按计划主动触发的回合 |
| ACP / 本地代理 | 更高级的本地代码或代理路径 |

这类来源区分的目的，不是让你背概念，而是让不同入口的边界不要混在一起。

## 为什么这件事重要

如果没有来源区分，远程入口、本地入口和持续任务入口就容易被混成同一种请求。

对普通用户来说，保留这层区分，最直接的意义有三点：

- 本地使用和远程使用不会被混成同一种入口。
- 审批、提示和回执可以根据来源保持不同强度。
- 当系统需要记录或回看时，更容易判断这次动作属于哪一类场景。

## 普通用户真正需要记住什么

你不需要记住“执行者”和“来源标记”的内部实现。更实际的理解方式是：

- 系统知道这次动作是不是你在侧边栏里发起的。
- 系统也会区分它是不是远程消息，或者是不是定时流程触发的。
- 这种区分有助于审批、回执和边界控制。

Ghast AI 当前会区分本地、远程和持续任务等不同来源路径。对普通用户而言，这种来源区分的意义在于保留更清晰的使用边界、审批边界和责任边界，而不是把所有请求都视为同一种输入。
