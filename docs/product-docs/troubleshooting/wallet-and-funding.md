# 钱包与充值问题

## 本页说明

本页用于处理钱包始终锁定、导入失败、模型充值不成功，或退款额度和预期不一致等问题。

## 先判断问题更像哪一类

| 现象 | 更适合先检查什么 |
| --- | --- |
| 钱包总是锁定，无法继续操作 | 当前是否已经正确解锁，自动上锁时间是否过短 |
| 无法创建或导入钱包 | 输入内容是否完整、格式是否正确 |
| 钱包里有钱，但模型页面仍提示需要充值 | 先区分钱包余额和模型可用资金不是同一层 |
| 充值失败 | 钱包余额是否足够支付充值和链上手续费 |
| 退款额度比预期小 | 先确认当前可退范围，而不是把全部余额都当成立刻可退 |

## 标准排查顺序

1. 先确认钱包已经解锁。
2. 再确认钱包所在网络和余额状态符合当前充值需求。
3. 再回到模型页面确认当前最低充值要求，以及这次看到的是哪一层余额。
4. 如果是退款问题，先确认当前显示的是可退款额度，而不是把全部余额都按立刻可退来理解。

## 不要先怎么判断

- 不要把钱包余额和模型资金当成同一层金额。
- 不要在钱包还没解锁前，就直接判断充值路径失效。
- 不要把“暂时不能全部退款”直接等同于异常。

## 仍然无法恢复时

如果你已经确认钱包可正常解锁、余额足够、当前模型的充值条件也满足，但充值或退款仍然异常，再继续回到钱包页或模型页核对具体状态会更合适。

钱包与模型资金是两层不同的资金路径。对普通用户而言，排查这类问题时，应先确认钱包状态和余额，再确认模型资金路径；退款则应优先按当前可退范围理解，而不是把页面全部余额都视为立刻可退。
