# 登录与激活问题

## 本页说明

本页用于处理登录成功但仍无法进入正常使用状态、一直停在邀请码步骤，或登录状态反复失效等问题。

## 先判断是不是这类问题

| 现象 | 更像哪类问题 |
| --- | --- |
| 已完成 Google 登录，但仍进不了主界面 | 激活尚未完成，或当前会话没有正常恢复 |
| 页面一直停在邀请码输入页 | 邀请码未通过，或当前账号仍需激活 |
| 刚登录完成又被要求重新登录 | 登录会话没有正常保持 |
| 不确定这是不是出错 | 先把邀请码步骤视为正式使用路径的一部分 |

## 标准处理顺序

1. 先重新完成一次 Google 登录。
2. 如果页面继续要求邀请码，就先完成邀请码验证，不要把它当作异常步骤跳过。
3. 登录和邀请码都已完成后，再确认是否已经进入后续引导或主界面。
4. 如果仍然反复跳回登录或激活页，再按登录状态异常处理。

## 不要先怎么处理

- 不要在看到邀请码页时，先假设“本来应该自动跳过”。
- 不要把登录成功直接等同于账号已经完全可用。
- 不要在登录与激活还没完成前，就把钱包、模型或 Companion 问题混在一起排查。

## 仍然无法恢复时

如果你已经重新完成登录，邀请码也确认无误，但仍然无法进入正常界面，说明问题仍停留在登录或激活阶段，应继续优先处理这一层，而不是先转去排查后续功能。

Ghast AI 当前将登录与激活视为连续但不同的两步流程。对普通用户而言，邀请码步骤属于部分账号的正式使用路径；只有在登录与激活都完成后，账号才会进入完整使用状态。
