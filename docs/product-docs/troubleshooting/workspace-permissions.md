# 工作区权限问题

## 本页说明

本页用于处理工作区明明已经连接，却仍然无法读写目录、工作区权限失效，或当前目录范围不够用等问题。

## 先判断问题更像哪一类

| 现象 | 更适合先检查什么 |
| --- | --- |
| 已选择文件夹，但仍然无法访问本地文件 | 工作区是否真的连接成功，或授权是否已经失效 |
| 之前能用的工作区后来不可用 | 浏览器是否要求重新授权，或目录位置是否已经变化 |
| 操作提示超出工作区范围 | 当前目标是否本来就不在已连接目录内 |
| 想访问另一个项目目录 | 当前更需要的是更换或补充工作区，而不是直接放宽整机范围 |

## 标准排查顺序

1. 先确认 Companion 已被扩展正常识别。
2. 再确认当前工作区是否仍然处于已连接状态。
3. 如果浏览器要求重新授权，就重新选择对应文件夹并完成授权。
4. 如果当前目录范围不够，再优先更换工作区或补充工作区。
5. 只有在这些方式都不够时，才再评估是否需要放宽到 `full` 模式。

## 不要先怎么处理

- 不要把工作区失效直接理解成必须放宽整机边界。
- 不要在还没重新授权前，就先判断 Companion 本身异常。
- 不要把“当前目录不在工作区里”误判成文件权限完全失效。

工作区问题首先应被当作连接范围或重新授权问题处理。对普通用户而言，推荐顺序始终是先确认 Companion 已识别、再确认工作区状态、再重新授权或更换目录；只有在明确需要更大范围时，才进一步评估 `full` 模式。
