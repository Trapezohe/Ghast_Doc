# MCP Servers

## 本页说明

本页说明在 Ghast AI 里应该怎样理解 MCP Servers，以及普通用户在什么情况下需要关心这层能力。

## 先怎么理解这项能力

对普通用户来说，MCP Servers 不需要先按协议或配置项来理解。更容易理解的方式是：

它让 Ghast AI 在接入 Companion 后，可以进一步拿到某些扩展本身不具备的本地工具能力。

这些能力可能包括：

- 文件系统访问
- 搜索类工具
- 数据库类工具
- 浏览器自动化类工具

也就是说，MCP 的重点不是“多一套技术名词”，而是“让某些更深入的本地工具真正可用”。

## 什么时候你通常不需要关心它

如果你的使用场景主要还是：

- 日常聊天
- 页面理解
- 钱包和模型的基础使用
- 记忆和技能的普通使用

那么当前通常不需要优先关心 MCP Servers。

对大多数普通用户来说，先把扩展、Companion、工作区和基础使用路径跑通，比一开始就进入 MCP 更重要。

## 什么时候它会变得重要

通常只有在下面这些情况里，MCP Servers 才会真正成为重点：

- 你需要更强的本地工具能力
- 某些技能或高级使用路径明确依赖额外工具
- 你希望 Ghast AI 在本地环境里处理更深入的任务
- 你已经完成基础接入，准备继续扩大本地能力深度

换句话说，MCP 更适合出现在“继续深入使用”这一阶段，而不是“第一次开始使用”这一阶段。

## 它和 Skills 的关系

对普通用户来说，最简单的理解方式是：

- Skills 更像“告诉 Ghast AI 应该怎么做”
- MCP 更像“让 Ghast AI 真正拿到某些额外工具”

因此，两者经常一起出现，但并不是同一件事。

有些技能只靠方法就能工作；有些技能要真正发挥作用，还需要 Companion 和 MCP 提供更深入的本地工具支持。

## 使用前先确认什么

如果你准备继续使用这层能力，建议先确认下面几件事：

1. Companion 已经安装完成，并且扩展能够正常识别。
2. 当前不是安装放行、自动配对或基础连接问题。
3. 你已经连接了明确工作区，并优先保持 `workspace` 模式。
4. 你现在的目标，确实需要额外本地工具，而不只是普通聊天或页面能力。

这样做的目的，是先把基础接入和本地边界确认清楚，再进入更深入的工具层。

## 如果相关 MCP 能力没有按预期工作

如果你已经进入这一步，但相关工具能力仍然不可用，建议按下面顺序处理：

1. 先确认 Companion 当前已经正常连接。
2. 再确认问题是不是其实出在工作区边界或权限模式。
3. 如果仍然异常，再进入[诊断与自检](diagnostics-and-self-check.md)，查看本机环境和相关工具可用性。
4. 如果问题更像安装、识别或自动配对失败，再查看[Companion 连接问题](../troubleshooting/companion-connection.md)。

也就是说，不要把所有高级能力异常都直接归因于 MCP；很多时候，问题其实发生在更前面的安装、识别或本地边界阶段。

在 Ghast AI 当前产品中，MCP Servers 属于 Companion 之上的更高级本地工具能力层。对普通用户而言，它主要服务于需要更深本地工具支持的技能和高级使用路径；只有在基础接入已经完成、且明确需要额外工具能力时，才需要把它作为重点理解和使用对象。
