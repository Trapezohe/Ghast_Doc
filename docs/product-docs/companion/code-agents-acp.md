# Code Agents / ACP

## 本页说明

本页说明 Ghast AI 中更高级的本地代码与自动化能力应该怎样理解，以及普通用户在什么情况下才需要进入这条能力路径。

## 先怎么理解这项能力

对普通用户来说，Code Agents / ACP 不需要先按内部协议来理解。更容易理解的方式是：

当你不只是希望 Ghast AI 在聊天中给出建议，而是希望它在本地工作区里持续处理更深入的代码或自动化任务时，就会进入这条高级使用路径。

当前这类能力依赖 Companion 在本机拉起本地代码代理会话，并用于承接更重的本地任务。

## ACP 对普通用户意味着什么

对普通用户而言，ACP 可以理解成这类本地代码代理会话的运行与接入方式。

你通常不需要单独学习这套机制，只需要知道两点：

- 它用于支撑更深入的本地代码代理任务
- 它不是首次上手时必须单独配置的一层

也就是说，用户真正需要关心的不是协议本身，而是自己是否已经进入需要本地代码代理的使用阶段。

## 什么时候这条能力会变得重要

通常只有在下面这些情况里，Code Agents / ACP 才会成为重点：

- 你希望 Ghast AI 参与更深入的本地代码任务
- 你已经准备围绕某个工作区长期使用 Ghast AI
- 你希望它处理比普通命令更重、更持续的本地任务
- 你已经完成基础接入，准备把产品用到更深入的本地代码与自动化阶段

如果你当前只是日常聊天、页面理解、基础工作区使用或普通命令任务，这一页通常还不是优先项。

## 什么时候你还不需要关心它

更稳妥的判断方式是：只要下面这些基础路径还没有跑顺，就还不需要优先进入 Code Agents / ACP：

- Companion 还没有稳定识别
- 工作区还没有连接完成
- 命令运行时和基础本地能力还没有稳定可用
- 你现在的目标还停留在普通问答、页面理解或轻量本地任务

先把基础能力用顺，再进入这条更重的本地代理路径，通常会更符合实际使用节奏。

## 使用前先确认什么

如果你准备使用这类能力，建议先确认下面几件事：

1. Companion 已经安装完成，并且扩展能够正常识别。
2. 当前不是安装放行、自动配对或基础连接问题。
3. 你已经连接了明确工作区，并优先保持 `workspace` 模式。
4. 命令运行时和其他基础本地能力已经能够正常工作。
5. 你现在的任务，确实需要更深入的本地代码或自动化处理，而不是普通聊天就能解决。

这套顺序的目的，是先把基础接入、本地边界和工作区路径确认清楚，再进入更重的本地代理阶段。

## 日常使用时更适合怎么理解

对普通用户来说，这类能力更适合被理解成：

- 不是基础上手入口，而是更深入的高级路径
- 不是替代所有普通对话，而是处理更重的本地代码任务
- 不是默认全开放能力，而是建立在 Companion、工作区和本地边界之上的进一步能力层

因此，Code Agents / ACP 更适合在你已经明确需要它时再使用，而不是为了“把能力全开”而提前进入。

## 如果这类能力没有按预期工作

如果你已经进入这一步，但本地代码代理能力仍然不可用，建议按下面顺序处理：

1. 先确认 Companion 当前已经正常连接。
2. 再确认工作区、权限模式和命令运行时是否已经正常工作。
3. 如果仍然异常，再进入[诊断与自检](diagnostics-and-self-check.md)。
4. 如果问题更像安装、识别、MCP 或高级能力环境问题，再继续查看相关故障页。

也就是说，不要把所有高级能力异常都直接归因于 Code Agents / ACP；很多时候，问题实际上发生在更前面的安装、连接、工作区或基础本地能力阶段。

Code Agents / ACP 属于 Ghast AI 当前更高级的本地代码与自动化能力路径。对普通用户而言，它应被理解为建立在 Companion、工作区和基础本地能力之上的进一步能力层，更适合已经完成基础接入，并明确需要更深入本地代码任务的用户使用。
