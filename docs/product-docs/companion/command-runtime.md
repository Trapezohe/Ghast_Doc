# 命令运行时

## 本页说明

本页说明在 Companion 已经安装并完成接入之后，Ghast AI 如何使用本地命令能力，以及这项能力在日常使用中应如何理解。

## 这项能力适合什么时候使用

命令运行时更适合下面这些场景：

- 围绕明确工作区执行本地任务
- 需要运行脚本、命令或本地工具
- 需要让 Ghast AI 处理浏览器本身无法完成的本地步骤

如果你的当前需求主要还是聊天、页面理解、记忆、钱包和基础模型使用，那么这项能力通常还不是你最需要优先理解的部分。

## 先怎么理解这项能力

对普通用户来说，命令运行时不应被理解成“把整台机器默认交给助手执行”。

更准确的理解方式是：

- 它是 Companion 提供的一项可选本地能力
- 它更适合围绕你已经连接的工作区使用
- 它的可用范围会受到当前权限模式影响
- 较高风险的命令当前会在扩展里要求用户确认

也就是说，命令运行时的重点不是“能不能无限执行”，而是“在什么边界内，为哪些本地任务提供帮助”。

## 使用前先确认什么

如果你准备使用命令运行时，建议先确认下面几件事：

1. Companion 已经安装完成，并且扩展能够正常识别。
2. 当前问题不是安装放行或自动配对问题。
3. 你已经连接了明确的工作区。
4. 当前优先保持在 `workspace` 模式，而不是直接放宽到 `full`。

这套顺序的目的，是先把安装、识别和工作区边界确认清楚，再进入命令执行阶段。

## 日常使用的推荐理解方式

对大多数用户而言，命令运行时更适合按下面方式使用：

- 先围绕一个明确工作区展开
- 先处理具体、有限的本地任务
- 先保持默认更收敛的本地边界
- 只在确有必要时再放宽本机范围

如果你的任务本身并不需要本地命令参与，就没有必要为了“功能更全”而提前打开更大的本地范围。

## 不要怎么理解这项能力

更稳妥的做法，是避免下面这些常见误解：

- 不要把命令运行时理解成默认无边界的本机控制
- 不要在还没有明确工作区时，就把它当作主要使用入口
- 不要把权限模式放宽，当成使用命令运行时的前提条件
- 不要把安装、识别和命令执行问题混在一起处理

如果当前连 Companion 还没有被扩展识别，那么应先处理连接问题，而不是直接回到命令运行时本身。

## 如果命令能力没有按预期工作

如果你已经安装 Companion，但命令能力仍然不可用，建议按下面顺序排查：

1. 先确认扩展已经识别 Companion。
2. 再确认当前工作区和权限模式设置是否符合预期。
3. 如果仍然异常，再进入[诊断与自检](diagnostics-and-self-check.md)。
4. 如果问题更像安装、识别或自动配对失败，再查看[Companion 连接问题](../troubleshooting/companion-connection.md)。

这样做的目标，是先分清问题属于安装、连接、边界还是命令执行，而不是直接把所有异常都归因于命令运行时。

Ghast AI 当前的命令运行时属于 Companion 提供的高级本地能力。对普通用户而言，它应被理解为围绕工作区展开的本地命令能力，而不是默认面向整台机器开放的执行能力；在标准使用路径下，应优先保持 `workspace` 模式，并在较高风险命令需要时通过扩展确认后再继续执行。
